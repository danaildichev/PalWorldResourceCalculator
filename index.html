Search & Filter
  • The recipes column shows you resources that can be crafted.
  • "Atomic" resources, like Stone and Wood, do not appear in this list because they do not have a recipe.
  • Type any part of the name of the resource you are looking for.
  • You can clear the search results by clicking on the button with the icon.
  • Hover over "Filter" to show the resource groups dropdown.
  • Select a resource group to limit which resources appear in the recipes list.
List Items
  • Hover over a resource to show its "Info" and "Add to Queue" buttons.
  • "Info" Button:
    • The "Resource Info Modal" will show you details about that resource and two tabbed panels with additional information.
    • If that resource has a recipe, the "Recipe" tab will show you which ingredients and how many are required to make 1 unit.
    • If that resource is used as an ingredient for other craftable resources, the "Used In" tab will show you which ones.
    • You can click on any resources listed in the "Recipe" and "Used In" tabs to see details about that resource.
    • The "Resource Info Modal" will also allow you to add or remove an item from your Queue.
  • "Add to Queue" Button:
    • Pressing this button will send that resource to your Queue.
    • You are ready to start using the Queue.

With our data set we found out that only 61 items are used as ingredients.

Using List Items
  • Use the drag handle to re-order your Queue.
  • Click the drag handle to select one or more items and move them together.
  • The quota field is for controlling how many units you want to make.
    • The Resource Calculator will update the queue item's UI to reflect changes.
  • Use the expand/collapse button to toggle the ingredients list.
  • The progress bar indicates how close your inventory is to covering the sum of required ingredient quantities.
  • Use the inventory input field to record how much of each ingredient you have.
    • The Resource Calculator will update the queue item's UI to reflect changes.
    • The Resource Calculator will also update the UI of any other queue items that have that ingredient in their recipe.
    • These changes are synced with the Inventory modal.
  • A recipe that has only 1 ingredient will not display a summary row.
Search & Filter
  • Use the filter buttons to limit which resources are shown in the Inventory modal's grid.
  • Use the search bar to find a particular resource.
    • Type any part of that resource's name.
Grid Items
  • Use the inventory input field and/or the decrement/increment buttons to change the amount of inventory you have for that resource.
  • Changes made here are automatically applied to any Queue items that have that resource in their ingredients list.
  • You can click on the name or thumbnail of a resource to focus in on the input field.
Actions
  • Clear the contents of the note area.
  • Copy the contents of the note area to your clipboard.
  • Append content from your clipboard to the notes area.
  • Save your notes as a text file.
  • Print your notes.
Content Area
  • You can type anything you'd like here.
  • If you do not see a blinking cursor, it means the content area is not in focus.
  • When the content area is not in focus it will have a background color that is different than the Notes modal.
Option 1: The "Save" Button
  • Press this button to turn the state of your data into a file.
  • Depending on what system and browser you are using this will result in a file picker or a direct download.
  • Navigate to the folder where you installed this app, for example
    • C:\xampp\htdocs\PalWorldResourceCalculator
  • Find the corresponding JS file in <appPath>\resource-calculator\src\JSON
  • Then overwrite the file.
  • The next time you (re)load this app, it will initialize with the data you just saved.
Option 2: The "Get" and "Cut" Buttons
  • Press the "Get" button to put your app data in the corresponding textarea as JSON.
  • Press the "Cut" button to extract it from the textarea and write it to your clipboard.
  • Navigate to the folder where you installed this app, for example
    • C:\xampp\htdocs\PalWorldResourceCalculator
  • Find the corresponding JS file in <appPath>\resource-calculator\src\JSON
  • Then overwrite the file.
  • The next time you (re)load this app, it will initialize with the data you just saved.
More Details

resources.js

  • Do not make edits.
  • Contains a few details about resources.

recipes.js

  • Do not make edits.
  • Stores a resource's ingredients and their required quantities.

inventory.js

  • Make careful edits.
  • Stores a resource's inventory level.
  • All resource IDs from "resources.js" must be listed.
  • All inventory amounts must be 0 or more.

queue.js

  • Make careful edits.
  • Stores queue items, their quota, and represents the order in which queue items will appear.
  • At a minimum that file must export {}
  • If you wish to write out your queue items they must be written within the curly braces as:
    "resourceID": quotaAmount, "resourceID": quotaAmount, etc...
    • resourceID must be wrapped in double quotes.
    • resourceID must be a valid ID from "resources.js"
    • quotaAmount must be a number (no double quotes)
    • quotaAmount must be 1 or more.

favorites.js

  • This file is not in use. Like a couple of other ideas, this could be a version 2 kind of thing.
